# Code of Conduct

MBS Terminal is a small open source project, and contributors are expected to keep the project useful, respectful, and welcoming.

## Expected Behavior

- Be kind and patient with people asking for help.
- Assume good intent while still giving clear technical feedback.
- Keep issues and pull requests focused on improving the project.
- Respect different skill levels, operating system setups, and workflows.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Discriminatory language or conduct.
- Publishing private information without permission.
- Repeated off-topic comments after being asked to stop.

## Reporting

If you see behavior that makes the project unsafe or unwelcoming, report it through a GitHub issue with only the minimum detail needed to start the conversation. If private vulnerability reporting is enabled for the repository, use that for sensitive safety issues.

## Enforcement

Maintainers may edit or delete comments, close issues, block users, or take other reasonable action to protect the project and its contributors.
